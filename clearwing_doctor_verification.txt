    # Check timeout during directory traversal
    if start_time is not None and timeout_seconds is not None:
        elapsed = time.time() - start_time
        if elapsed > timeout_seconds:
            logger.warning(
                "SourceAnalyzer._iter_source_files: timeout reached after %.1fs during directory traversal at %s",
                elapsed, dirpath
            )
            return  # Exit the generator early
    current_depth = dirpath.rstrip(os.sep).count(os.sep) - root_depth

## 5. Implementation Contract

**Intended mechanism:** Add timeout check inside `os.walk()` loop to prevent process from being killed by SIGKILL during extensive directory traversal.

**Data flow:**
1. `analyze()` records start time and calls `_iter_source_files()` with timeout parameters
2. `_iter_source_files()` checks elapsed time at each directory iteration
3. If timeout exceeded, generator exits early with warning log
4. `analyze()` receives timeout signal and sets `result.timed_out = True`

**Invariants:**
- File iteration terminates within configurable time bounds
- Timeout check does not affect analysis quality
- No files are silently skipped
- Progress is logged for debugging

**Forbidden shortcuts:**
- ❌ Do NOT silently skip files or directories when limits are hit
- ❌ Do NOT reduce analysis quality to meet time limits
- ❌ Do NOT add TODO stubs or placeholder implementations
- ❌ Do NOT hardcode paths or make assumptions about directory structure
